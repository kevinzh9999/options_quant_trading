# 期权策略开发框架

> 2026-04-06 | 基于当前系统架构和数据资产

---

## 一、当前期权持仓现状

**持仓**
- MO2606-C-8500 空16手 → theta收割 + 看跌方向
- MO2606-P-6000 空16手 → theta收割 + 看涨方向

**交易方式**：纯手动。凭判断选行权价、选时机、选手数。没有系统化的入场/出场/仓位管理规则。

**收益来源**：
1. Theta衰减（每天约+6000元）
2. 方向性判断（远虚值=赌不到）
3. IV均值回归（高IV时卖出，IV回落时赚vega）

**问题**：
- 没有系统化的开仓信号（什么时候卖、卖哪个行权价、卖多少手）
- 没有系统化的平仓/止损规则
- Greeks管理靠盘感不靠数据
- 和日内期货策略没有协调机制

---

## 二、期权策略的分类框架

按风险收益特征分四大类：

| 类型 | 特征 | 策略 | 收益来源 |
|------|------|------|---------|
| **方向性策略** | 对标的方向有明确判断 | 买Call/Put、垂直价差 | 标的涨跌 |
| **波动率策略** | 对波动率方向有判断，对标的方向中性 | Straddle、Strangle、Butterfly | IV vs RV差异(VRP) |
| **Theta策略** | 卖时间价值，赚取确定性最高的theta衰减 | 卖远虚值Put/Call、Iron Condor | theta衰减（尾部风险大） |
| **结构性策略** | 利用市场定价偏差(skew、期限结构) | 日历价差、比率价差、风险逆转 | 定价偏差修复 |

**我们当前做的是**：Theta策略（卖远虚值）+ 隐含的方向判断（选行权价时隐含了对涨跌范围的判断）。

## 三、和日内期货策略的关系

### 互补性分析（基于215天数据）

| 市场环境 | 日内期货PnL | 期权卖方PnL | 组合 |
|---------|------------|------------|------|
| 高振幅震荡（9月、3月） | 大赚 +834pt/月 | 有风险（gamma损失大） | 期货主导 |
| 低振幅趋势（7-8月） | 微利 +6pt/天 | 稳定theta +6000元/天 | 期权主导 |
| 窄幅震荡（10月） | 亏损 -27pt/天 | 最佳环境（theta+IV回落） | 期权独撑 |
| 暴跌/暴涨（3/23） | 大赚（做空）+100pt | 大亏（put爆）-数万元 | 对冲但不完美 |

**核心矛盾**：期权卖方最怕的（暴跌/高波动）恰好是日内策略最赚的。理论上完美互补，但实际上暴跌日期权的亏损可能远大于期货的盈利（gamma非线性放大）。

### 协调原则

1. 期权仓位的delta不应和日内期货信号方向长期矛盾
2. 高波动期：减少期权卖方仓位，让日内策略赚钱
3. 低波动期：增加期权卖方仓位，弥补日内策略的不足
4. 极端行情：期权止损优先于theta收割

## 四、期权策略开发的五个层次

### Level 1：系统化当前手动操作（最低门槛，最高性价比）

把现在手动做的事情变成有规则的。

#### 卖方开仓规则

**什么时候卖？**
- IV分位 > P60（IV偏高时卖方有利）
- VRP > 0（IV > RV，卖方有统计优势）
- 已有数据：daily_model_output 中的 iv_percentile_hist 和 vrp

**卖哪个行权价？**
- delta < 0.15 的虚值期权（约1个标准差外）
- 安全距离 > 10%（当前价格到行权价的距离）
- 已有数据：vol_monitor 实时计算每个行权价的 delta

**卖多少手？**
- 总 vega 敞口 < 账户权益的 0.5%/1%IV变动
- 单腿不超过总持仓的 40%
- 保证金占用 < 账户权益的 40%

#### 卖方平仓规则

**止盈：**
- 期权价格跌到初始卖出价的 20% 以下（赚了 80% theta）
- 距到期 < 5天且 OTM（让剩余theta自然衰减）

**止损：**
- delta > 0.30（从深虚值变成中虚值，风险急增）
- 安全距离 < 5%（标的快要碰到行权价了）
- 浮亏 > 卖出收到权利金的 2 倍
- VRP < -5%（IV < RV，卖方统计劣势，市场在定价真实风险）

**调仓：**
- 距到期 < 10天：移仓到下月
- delta 从 0.10 变到 0.25：向更远虚值移仓

#### 买方规则

**什么时候买？**
- IV分位 < P20：提示"波动率历史低位，可考虑买入远月straddle"
- VRP < -3%：提示"卖方统计劣势，不新开卖方，可考虑买入保护"
- B2象限（高IV+负VRP+趋势）：提示"买入顺趋势方向的put/call"

**买方仓位管理：**
- 买方单笔投入 < 账户权益的 0.5%（亏完就亏完，当彩票）
- 买方持仓总市值 < 账户权益的 2%
- 买方不做时间价值衰减的对手——选到期日 > 30天的合约

#### 实现方式

不需要自动交易，在 morning_briefing 和 vol_monitor 中加入规则检查和提示。操作者看到提示后手动执行。

### Level 2：基于象限的仓位管理

利用已有的象限系统（IV x VRP x Hurst）动态调整期权仓位：

| 象限 | 环境 | 策略 |
|------|------|------|
| **A1**（高IV + VRP>0 + 震荡） | 最佳卖方环境 | 满仓卖方（delta<0.15），可靠近ATM(0.20-0.25)赚更多theta |
| **A2**（高IV + VRP>0 + 趋势） | 卖方可行但谨慎 | 半仓卖方，保持远虚值(delta<0.10)，趋势向下时多卖call少卖put |
| **B1**（高IV + VRP<0 + 震荡） | 等待 | 减仓到1/3，VRP<0说明市场波动比IV定价的还大 |
| **B2**（高IV + VRP<0 + 趋势） | 最危险 | 最小仓位或清仓，只保留最远虚值保护性仓位 |
| **低IV**（P<40） | 不卖 | IV太低theta太少，可考虑买方策略 |

实现方式：quadrant_monitor 已经在输出象限判断，加上对应的期权仓位建议。

---

### Level 3：波动率交易策略（VRP捕获）

不是简单的"卖期权收theta"，而是系统性地交易IV和RV的差异。

**核心逻辑**：
```
VRP = IV - RV（波动率风险溢价）

长期来看 VRP > 0（卖方有统计优势）
但短期VRP会大幅波动：
  VRP >> 0：IV远高于RV → 大力卖 → IV回归时赚vega
  VRP ≈ 0：中性 → 小仓位
  VRP << 0：RV远高于IV → 不卖，甚至买
```

**信号生成**：
```python
vrp = atm_iv - blended_rv
vrp_percentile = percentile_rank(vrp, history_60d)

if vrp_percentile > 80:   signal = "SELL_VOL_AGGRESSIVE"
elif vrp_percentile > 50:  signal = "SELL_VOL_NORMAL"
elif vrp_percentile > 20:  signal = "NEUTRAL"
else:                      signal = "BUY_VOL"
```

需要的数据：daily_model_output 已有 VRP 和 VRP 分位，可以直接用。

---

### Level 4：Skew和期限结构交易

**Skew交易**：
- RR极高（Put skew很大）→ 卖OTM Put + 买OTM Call（收skew溢价）
- RR极低 → 反向
- 需要：盘中5分钟RR数据（vol_monitor已在计算，需积累1个月干净数据）

**期限结构交易（日历价差）**：
- 近月IV > 远月IV（倒挂）→ 卖近月 + 买远月（收倒挂溢价）
- 近月IV < 远月IV（正常）→ 维持
- 需要：多月份的ATM IV（vol_monitor已有）

---

### Level 5：期权Greeks对冲 + 日内期货的联合策略

最高级：期权提供底仓收益（theta），日内期货做delta对冲和方向性交易。

```
期权底仓：卖Strangle → 每天收theta，净delta ≈ 0（方向中性）
日内期货：
  信号触发做多 → 期货开多 → 组合delta变正
  信号触发做空 → 期货开空 → 组合delta变负
  日内平仓后 → delta回到期权的中性值
效果：
  无信号时 → 纯theta收入
  有信号时 → theta + 期货方向性利润
  期货止损时 → theta部分弥补期货亏损
```

工程复杂度最高，需要实时的组合Greeks计算和delta对冲逻辑。

## 六、开发路线图

### Phase 1：系统化手动操作（1-2周）

目标：把 Level 1 的规则写成代码，在 briefing 和 vol_monitor 中显示。

- **1.1 开仓信号**：morning_briefing 中加入"期权开仓建议"，基于 IV分位/VRP/安全距离
- **1.2 持仓监控**：vol_monitor 中每个持仓的 delta/安全距离/止损条件检查（红/黄/绿灯）
- **1.3 平仓/移仓提示**：距到期<10天提醒移仓；delta>0.25 提醒止损；浮亏>2倍权利金止损

产出：纯提示，操作者手动执行。

### Phase 2：VRP策略历史回测（2-3周）

目标：验证"高VRP时卖、低VRP时不卖"是否在历史上有效。

- **2.1 回算历史IV曲面**：从 options_daily（7年）逐日回算 ATM IV，计算历史 VRP 时间序列
- **2.2 VRP策略回测**：每月初根据 VRP 分位决定卖方仓位大小，固定卖 delta=0.15 的 Strangle
- **2.3 风险分析**：最大回撤、尾部风险、和日内期货策略的相关性

产出：VRP策略是否有正期望、最优的VRP阈值。

### Phase 3：Skew交易研究（3-4周）

目标：利用 options_min（6年5分钟数据）研究盘中 Skew 信号。

- **3.1 历史Skew回算**：从 options_min 逐5分钟回算 25D RR
- **3.2 Skew信号研究**：Skew急变后的标的走势、极端值的预测力、和日内策略的信号重合度
- **3.3 如果有效**：接入 sentiment_mult 或作为独立信号

### Phase 4：联合策略（2-3个月后）

前提：Phase 1-3 完成 + 日内期货策略实盘稳定 2 个月+。

- 4.1 组合Greeks实时计算
- 4.2 delta对冲逻辑
- 4.3 联合回测
- 4.4 实盘联合运行

## 七、风险管理框架

### 期权特有的风险

| 风险 | 度量 | 限制 |
|------|------|------|
| Delta风险 | 净delta x 标的1%变动 | < 账户 2% |
| Gamma风险 | 净gamma x 标的1%变动^2 | < 账户 1% |
| Vega风险 | 净vega x 1%IV变动 | < 账户 3% |
| Theta | 正theta越大越好 | — |
| 尾部风险 | 标的跌20%时的损失 | < 账户 15% |

### Stress Test 场景

| 场景 | 影响 | 预估损失 |
|------|------|---------|
| 标的1天跌5%（类似3/23） | P6000还有20%安全距离→几乎不受影响 | < 3万 |
| 标的1周跌15%（极端熊市） | P6000距离只剩5%→delta急增gamma爆炸 | 30-50万（需精确计算） |
| IV暴涨10pp（恐慌事件） | vega=-21508 x 10 = -21.5万（theta部分对冲） | ~15-20万 |

### 风控规则

**硬限制（不可违反）：**
1. 总保证金占用 < 账户权益的 50%
2. 单一方向（纯多/纯空delta）< 账户权益的 3%/1%标的变动
3. 任何单腿浮亏 > 收到权利金的 3 倍 → 强制止损
4. B2象限（高IV+负VRP+趋势）→ 只减仓不加仓

**软限制（可调整）：**
5. IV分位 < P40 → 不新开卖方仓位
6. 距到期 < 7天 → 移仓到下月
7. 单腿delta > 0.30 → 移到更远虚值

## 八、核心认知

### 期权卖方的本质

不是"赚theta"——theta只是表象。本质是**卖保险**：收取保费（权利金），承诺在极端情况下赔付（标的大幅波动时亏钱）。

长期来看保险公司赚钱（VRP>0 = 保费 > 实际赔付），但单次极端事件可能巨亏。所以：
- **仓位管理比信号更重要**——不是"该不该卖"而是"卖多少"
- **止损纪律比theta收入更重要**——一次不止损可能吃掉半年theta
- **分散化**——多到期月、多行权价、Call+Put双边

### 和日内策略的本质区别

| | 日内策略 | 期权卖方 |
|---|---|---|
| 频率 | 高频，日内多笔 | 低频，月度调仓 |
| 仓位 | 小仓位（1手） | 大名义仓位（16手） |
| 盈利来源 | 胜率 x 频次 | 概率 x theta |
| 单笔亏损 | 有上限（止损0.5%） | **无上限**（gamma加速） |
| 最大风险 | 连续止损 x 仓位 | **尾部事件 x gamma** |

两者的风控逻辑完全不同，不能套用日内策略的止损思维管理期权。

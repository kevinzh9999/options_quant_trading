# Factor Research Methodology Lessons

> 基于 2026-04 横盘反转因子、threshold 全量验证、节奏摆动日策略研究的经验总结

## 核心原则

### 1. 小样本局部异常默认不可信

**教训**：219天数据上IM/IC [75,80)都显示MFE/MAE显著低于周边（0.75和0.95），跨品种一致看起来是强证据。投入大量时间做isolated_m假设、V=30子类分析等。900天数据上IM [75,80) MFE/MAE=1.20，陷阱完全消失。

**原则**：
- 任何基于<500笔样本发现的"局部异常"，在投入成因分析之前必须先用更大数据验证
- 跨品种一致不能作为小样本结论的证据——两个小样本的一致可能仍然是噪音
- MFE/MAE这类中位数指标对小样本敏感，90-130笔档位里少量outlier可以显著偏移结果

### 2. 先用大数据跑初始profiling

**教训**：先用219天发现、后用900天验证的流程，容易让研究员在初期对小样本噪音投入过多时间。

**原则**：
- 探索因子时默认**第一次就用全量数据跑**，而不是先用子样本发现再用全量验证
- 如果全量数据跑不动，至少保留30-50%作为"纯观察数据"，不参与假设形成
- 任何基于子样本的发现，在"纯观察数据"上独立验证，不通过就放弃

### 3. In-sample / Out-of-sample 分离是硬约束

**教训**：IM threshold扫描IS最优是thr=60、OOS最优是thr=55，两者不一致。如果只看IS会选错。

**原则**：
- 任何参数优化必须做IS/OOS分离，OOS段要有足够样本（>=200交易日）
- IS发现的最优参数在OOS上必须至少"不明显变差"才能上线
- 一旦OOS跑过就不能回头调IS参数再试，这会污染独立性

### 4. 单一forward period的IC可能严重低估因子强度

**教训**：横盘反转因子简化版在forward=6bar的Daily IC只有0.030，看起来弱。扫描forward=2-24后发现IM低振幅日在forward=5-15有稳定高原，峰值RankIC=0.341。

**原则**：
- 新因子评估时必须扫描forward period而不是只看[3,6,12]
- 反转因子有"兑现窗口"概念，太短未兑现、太长被抵消，需要找到窗口
- 峰值位置本身是因子特征，稳定在某个forward range的因子比飘忽不定的更可信

### 5. IC高但分组收益差弱的因子不能用

**教训**：复杂版横盘反转因子全局Daily IC=0.08-0.12，看起来很强。但regime分组后Pos-Neg收益差只有5-10bps，扣掉交易成本后无实用价值。

**原则**：
- 全局Daily IC只是方向性指标，不能决定因子是否能用
- 实用性判断必须看分组后的收益差（bps），Pos-Neg差<10bps的因子没有实盘价值
- 评估因子时IC和bps差要同时看，任一不合格都不能上线

### 6. 稳定高原 vs 孤立尖峰

**教训**：参数搜索中找到的"最优参数"可能是真信号的一部分（稳定高原），也可能是搜索到的偶然点（孤立尖峰）。

**原则**：
- 不要只看"最优点"，看"邻域是否都差不多好"
- 相邻3个参数点都保持同向同强度 = 高原 = 可信
- 只有一个参数点达标、前后都明显差的 = 尖峰 = 过拟合
- 对forward sweep、参数sensitivity、时间分段都适用

### 7. 因子在regime之间反向是可能的，不是矛盾

**教训**：横盘反转因子在IM低振幅日正向有效（+18bps），在高振幅日无效，在IC所有regime无效。

**原则**：
- 好因子不一定全局稳定，可能是条件稳定的——特定条件下强正向、其他条件下被明确识别为不适用
- 因子的"适用边界"跟"预测力"一样重要
- Regime classifier的作用不只是"用哪套参数"，还包括"哪些因子该用/不用/取反"

### 8. 证伪也是有价值的研究结果

**教训**：复杂版横盘反转被证伪、isolated_m假设被证伪、[75,80)陷阱被证伪——每一个都排除了错误方向。

**原则**：
- 研究价值不只是"发现能用的因子"，还包括"排除不能用的方向"
- 实证通过的因子大概只占探索假设的10-20%，证伪是常态
- 证伪要记录在档，避免重复走死胡同

## 研究流程checklist

开始新因子研究前过一遍：

1. [ ] 初始评估数据量>=500笔样本或1年以上？
2. [ ] 是否保留>=30%数据作为纯观察数据不参与假设形成？
3. [ ] 假设是否有先验经济/微观结构解释？还是纯数据挖掘？
4. [ ] 参数空间是否有限（<20个组合）？是否设置了"高原vs尖峰"判定？
5. [ ] forward_periods是否覆盖了合理的兑现窗口范围？
6. [ ] 评估指标是否包括IC和分组bps差两个维度？
7. [ ] 初始发现看起来很强时，是否计划在独立数据上做OOS验证？

7条都肯定回答=流程合格。任何一条缺失=风险点。

### 9. 区分"虚假信号"和"效应量高估"

**教训**：横盘反转因子简化版在219天数据上显示IM低振幅日有+18.8bps峰值，在900天数据上峰值降到+9.6bps。初次判定为"跟[75,80)陷阱区一样的小样本噪音"，但两者本质不同：

- **[75,80)陷阱区**（真正的虚假信号）：219天MFE/MAE=0.75，900天MFE/MAE=1.20。**方向翻转，形态完全不一致**。这是随机噪音在小样本下产生的局部异常。
- **横盘反转因子**（效应量高估）：900天上fwd=2-24所有点都是正值，单调递增从+1.3到+9.6bps，fwd=12-24形成平坦高原。**形态完整，方向一致，只是绝对值缩小一半**。这是真实信号的效应量被小样本高估。

**原则**：
- 判断小样本发现是否真实，不能只看"峰值是否消失"，要看**大数据上的形态是否保留**
- 真实信号：形态一致（单调性、方向性、连续性都保留）、绝对值可能缩小
- 虚假信号：形态改变（方向翻转、单调性消失、随机震荡）
- 两者处理不同：
  - 虚假信号 → 归档放弃，未来不再探索
  - 效应量高估 → 进入"观察池"，不上线但不放弃，等样本累积或新评估维度出现时重新评估
- 不要用单一门槛值（如"<10bps就放弃"）做硬切分。门槛是参考不是铁律，特别是当数值刚好在门槛附近（9-11bps）时，应该看形态而不是数字

**对研究流程的影响**：
- 评估结果应有三种状态：**有效**（强度充足+形态稳定，可考虑上线）、**观察池**（形态稳定但强度不足）、**证伪**（形态随大数据消失或翻转）
- 观察池因子需要有明确的"重新评估触发条件"，避免无限期拖延

### 10. 主观观察的"大空间"几乎总是被高估

**教训**：节奏摆动日研究中，上帝视角看K线觉得"peak→低点36pt空间很大"，但实际策略只能抓到6-8pt（抓取率20%）。入场等待（3bar无新高）损失65%空间，出场后反弹再侵蚀剩余利润。

**原则**：
- 人眼看K线会自动过滤噪音，只记住清晰的反转点——这不是实时可执行的
- 实时交易必须等确认信号，确认信号意味着入场延迟，延迟意味着空间损失
- 校准法则：**基于主观K线观察的盈利期望 × 0.2-0.3 才是现实估计**
- 做基本面分析时先算理论空间，再乘以0.2-0.3作为策略设计的目标上限

### 11. 反直觉结果先查bug再解释市场（Occam's Razor）

**教训**：节奏摆动日参数搜索发现"target越紧越差"（tgt=0.3胜率28%但触发率98%），第一反应是去解释"市场反复震荡"。实际原因是target定义bug——入场价已低于target位，触发target=亏钱。tgt=0.3时66%的交易是伪触发。

**原则**：
- 看到反直觉结果时，按顺序检查：
  1. 代码逻辑是否有bug（边界条件、符号方向、索引越界）
  2. 参数定义是否清晰无歧义（target基于peak还是基于entry_price？）
  3. 数据是否有问题（时间标准混淆、重复数据）
  4. **最后**才考虑市场结构解释
- 大多数"反直觉"的实证结果，根因是前三类问题中的某一个

**案例1**：节奏摆动日 target=0.3 胜率28%但触发率98%。原因：target定义bug，入场价已低于target位。

**案例2**：IC threshold sweep 五个值(45/50/55/60/65)结果完全相同（PnL +2020, 笔数1970, WR 45.8%）。
- 错误解释："IC所有信号得分都>=65"——但同一报告的MFE/MAE profiling显示[60,65)有374笔信号，直接矛盾
- 实际原因：多进程fork后全局变量修改不同步。主进程修改SYMBOL_PROFILES['IC']['signal_threshold']，但子进程fork后有独立内存副本，看不到修改。五次sweep实际跑了五次相同的thr=60回测
- 关键信号：**"完全相同的结果" = bug的最强指示信号**。任何参数sweep产生完全一致的结果都应该立刻怀疑参数没传进去
- 修复后结果正常：thr=45 +2312pt(2217笔) vs thr=60 +2020pt(1970笔) vs thr=65 +1616pt(1825笔)

### 12. 参数搜索之前必须做基本面分析

**教训**：节奏摆动日研究先做了64组合参数搜索（IS最优+8.9pt），OOS衰退后又做12组合出场优化。两轮搜索都失败。之后才做基本面分析，发现理论空间36pt、入场后可用13pt、反弹侵蚀13pt——这些数字在搜索之前就能算出来，能直接框定合理的参数范围。

**原则**：
- 参数搜索之前先回答三个问题：
  1. 市场理论最大空间多少？（peak→全天最低）
  2. 入场后可抓取空间多少？（入场点→第一波低点）
  3. 出场后反弹侵蚀多少？（低点→收盘反弹）
- 有了这三个数字，参数空间直接框定。target不可能比"入场后可抓取空间"更贪
- 基本面分析不需要回测，只需要统计K线本身的价格结构

### 13. "更聪明"的规则不一定更好

**教训**：节奏摆动日研究中，4个上下文感知出场规则（时间+幅度门槛、时间分段敏感度、回撤门槛+跌破保护、综合版）全部不如简单的固定target=0.8。CA1全样本+3.6pt vs 旧基线+7.5pt，OOS后半甚至-1.0pt。

**原因分析**：
- 简单规则对未知形态有鲁棒性——固定target在"持续下跌日"吃大肉（+50pt），在"反弹日"少赚或小亏
- 复杂规则在"持续下跌日"提前出场（+12pt），在"反弹日"表现更好（+12pt）——但前者的损失远大于后者的收益
- 2025-08-29从+50跌到+12，4/7从+50跌到+37——复杂规则砍掉了大肉

**原则**：
- 新规则必须在**所有典型场景**上至少保持简单规则的表现，不是只在目标场景上更好
- 用3-4个代表性日子做sanity check：慢速持续下跌、快速W底、止损日、大赚日
- 如果新规则在某类日子上严重退化（-30pt+），即使在目标日子上改善也不值得

### 14. 复杂规则是隐式的参数搜索

**教训**：上下文感知出场规则里每一个判定数字（6bar持仓门槛、0.4amp幅度门槛、2bar无新低）都是一个隐藏参数。虽然没有显式sweep，这些数字是通过"在sanity check日子上调到合理"来选定的——本质上是在少数样本上做了隐式参数搜索。

**原则**：
- 统计规则中的每一个magic number都是参数，即使它看起来像"常识"
- 一个有5个判定条件的规则 = 5维参数空间。5维空间里找到一个"在2-3个日子上表现好"的组合毫不困难
- 复杂规则的自由度高，必须用更严格的OOS验证标准来补偿
- 规则越简单、自由度越低、越不需要担心过拟合

### 15. 路径并行机制需要明确互斥关系

**教训**：对称逻辑让"横盘衰竭"和"急速反转"两条入场路径并行触发。结果急速反转（X=0.15%阈值太低）97%先触发，横盘衰竭路径形同虚设。两种机制不是互补而是冲突。

**原则**：
- 设计多路径机制前，先确认两条路径的触发条件是否互斥
- 如果路径A的触发条件是路径B的子集（A几乎总是先触发），那路径B永远不会被用到
- 正确做法：要么用互斥条件（if A then skip B），要么用不同的触发维度（A看时间，B看幅度）
- 测试时必须看每条路径的实际使用率。使用率<10%的路径 = 死代码

### 16. 接受天花板是有价值的研究结论

**教训**：节奏摆动日研究经过12轮迭代（理想化测试、参数优化、bug修复、对称逻辑、上下文感知），最终确认+5.6-8pt是物理天花板。不是策略设计缺陷，而是市场结构决定的——入场等待损失65%空间，出场后反弹侵蚀剩余利润。

**原则**：
- 天花板的确认需要穷尽主要方向：入场优化、出场优化、入场+出场组合优化
- 一旦3个以上独立方向都撞到同一水平，接受天花板
- 天花板研究的价值：避免未来在相同方向上浪费时间
- 记录天花板的具体数字和原因，6个月后市场环境可能变化，可以用同样的框架快速重评

### 18. 多进程回测的参数必须通过args传给子进程

**教训**：IC threshold sweep用`multiprocessing.Pool`并行回测，主进程修改`SYMBOL_PROFILES['IC']['signal_threshold']`后启动子进程。但fork后子进程有独立的内存副本，主进程的修改不可见。五个threshold值跑出完全相同的结果。

**错误模式**：
```python
SYMBOL_PROFILES['IC']['signal_threshold'] = new_value  # 主进程修改
with Pool() as pool:
    results = pool.map(backtest_func, days)  # 子进程看不到修改
```

**正确模式**：
```python
def backtest_func(args):
    td, thr = args
    SYMBOL_PROFILES['IC']['signal_threshold'] = thr  # 子进程内部设置
    return run_backtest(td)

with Pool() as pool:
    results = pool.map(backtest_func, [(td, new_value) for td in days])
```

**检查方法**：
- sweep跑完后如果所有参数值的结果完全相同 → 立刻怀疑此bug
- 跑sweep前先用单进程版本跑2个不同参数，确认结果不同
- 子进程里加print验证收到的参数值

**适用范围**：所有依赖全局配置dict（SYMBOL_PROFILES等）的多进程代码。

### 19. IS/OOS反转指示市场结构变化，默认相信OOS

**教训**：IC threshold重验证(2026-04)中：
- IS段(600天): thr=45最优+2312pt, thr=60最差+2020pt
- OOS段(300天): thr=60最优+1732pt, thr=45次之+1645pt
- 最近30天: thr=60 +276pt vs thr=45 -63pt（方向完全反转）

IS说"降threshold"，OOS说"保持或升threshold"——**IS/OOS反转**。

**反转 vs 衰退的区别**：
- **OOS衰退**：IS最优在OOS仍最优，但绝对值降低。正常现象，IS最优仍可上线
- **OOS反转**：IS最优在OOS变成次优或最差。危险信号，IS最优不能上线

**诊断含义**：
- 反转不是参数过拟合（过拟合通常是衰退不是反转）
- 反转不是统计噪音（噪音下OOS排序接近随机，不会清晰反向）
- 反转指示市场结构在IS/OOS边界附近发生了真实变化

**操作规范**：
- 看到IS/OOS反转时，**默认相信OOS**（更接近未来市场环境）
- IS/OOS反转 = 明确的"不改动当前值"信号
- 在所有per-symbol优化中，"是否存在IS/OOS反转"作为必检项目

### 20. Per-symbol优化结论不能跨品种复制

**教训**：IM threshold 50→55成功(OOS +367pt)，直接对标IC预期类似改善。实际结果完全相反：
- IM: 降threshold去掉负期望边界交易 → IS+OOS一致改善
- IC: 降threshold在IS有改善但OOS反转 → 当前值60已是OOS最优
- IC的profiling显示信号质量不单调：[55,60) MFE/MAE=0.95(负)但[50,55) MFE/MAE=1.21(正)
- 这跟IM的"边界区间单调负期望"完全不同

**原则**：
- "对标某品种"只能作为研究起点（确定sweep范围），不能作为结论
- 每个品种必须独立做完整IS/OOS验证
- 结果预期应该比对标品种更宽（IM +367pt → IC预期 -200到+500pt）
- 接受"某品种不需要这个优化"作为合法结论
- 发现IC不需要降threshold，跟发现IC需要降threshold**同等有价值**——前者避免了~700pt的损失

**物理含义**：不同品种的市场参与者结构、流动性、机构占比不同。同一个signal_v2评分系统对不同品种的"信号质量分布"可以是完全不同的形态。

### 21. 相关性强度 ≠ 规则可操作性

**教训**：Score衰减vs PnL的Pearson相关IM r=-0.437, IC r=-0.508，统计学上是"强相关"。转化为出场规则后净亏-1712pt(IM)和-598pt(IC)。完全失败。

**转化效率的三个杀手**：
1. **触发延迟**：score衰减领先价格，但等到score跌20分时价格已经反向5-10pt。触发的74%时点上价格已浮亏——"在低位平仓"
2. **误触发的不对称代价**：正确触发收益(+11.3pt/笔)远小于误触发代价(-19.5pt/笔)。57%正确率无法覆盖1.7倍的代价不对称
3. **交易成本**：触发越多成本越高，低收益边际上成本吃掉所有改进

**经验法则**：
- r > 0.3：有信息但作为硬触发大概率失败
- r > 0.5：仍不够，转化效率通常<30%
- r > 0.8：可能可以作为硬触发
- 低相关信号应用作风险标签或仓位调整，不是硬触发

**事前估算**：测试前先算所需最低正确率 = P/(P+L)，其中P=误触发代价，L=正确触发收益。r=-0.5约对应60-65%正确率。如果所需正确率>75%，相关性必须>-0.7才有希望。

---

### 22. 当应该触发和误触发无法区分时，规则就是赌博

**教训**：Score衰减规则IM 917笔触发，57%应该触发/43%误触发。两组在所有可观测特征上几乎完全相同：
- entry_score: 69.4 vs 69.9
- M/V/Q分: 39.8/15.8/10.8 vs 40.5/15.9/10.5
- 多空方向: 53/47% vs 49/51%
- 即使"score跌>50"的严格条件下正确率也只有61%

**原则**：
- 任何候选规则必须做"两类拆解"：应该触发 vs 误触发
- 在5-10个可观测特征上对比两组
- 两组特征几乎相同 → 规则不可分 → 即使总体好也是过拟合
- 某特征能区分 → 加该特征作为附加条件
- 不可分但相关性真实 → 信号在分布层面成立但个体层面失效 → 用作风险标签，不用作硬触发

### 23. 负面结果发表前做最后一公里调查

**教训**：Score衰减出场第一轮模拟所有候选比基线差几千pt，CC直接判定R3。但报告有未解疑点：基线变了(IC从+3874跌到+1725)、触发统计未验证。操作者要求最后调查后发现：
- 基线bug：399笔短持仓trade被>=3bar过滤误删，丢了+2027pt
- 意外副产品：IM/IC短持仓trade表现完全相反（IM -2.7pt/笔, IC +2.4pt/笔）
- R3仍成立，但调查产生了新研究方向

**操作原则**：
- 失败结果发表前做"尽职调查"：基线一致性、内部数字对账、子组拆解、副产品识别
- 接受失败前问："如果再花1小时调查，可能改变结论吗？"
- 放弃真实失败的成本=0，放弃误判失败可能错过重要发现

### 24. IS/OOS必须不重叠，且IS必须跟训练窗口对齐

**教训**：IC threshold重验证用"IS 600天/OOS 300天"的时间对半切，但所有参数(trailing=2.0, dm=1.1/0.9, me=0.12)实际训练在最近219天上。219天落在OOS 300天内(73%的"OOS"实际是IS)。结果：
- "OOS"300天日均+6.0pt（被训练数据污染）
- 真正的OOS 681天日均+2.4pt
- IS 219天日均+9.8pt
- IS是真OOS的4倍——经典过拟合信号，但被错误的IS/OOS标签掩盖了

**正确做法**：
1. 明确训练窗口的起止日期
2. IS = 训练窗口（参数在这段数据上优化）
3. OOS = 全量数据 - 训练窗口（从未参与优化的数据）
4. 两者严格不重叠

**如果训练窗口是"最近N天"**：
- IS = 最近N天，OOS = 更早的所有数据
- OOS是历史不是未来——OOS好说明参数对历史也适配，但不保证未来
- OOS差有两种可能：(a)过拟合IS (b)市场结构真实变化
- 需要结合其他证据（月度PnL趋势、参数稳定高原等）区分

**受影响的研究**：
- IC threshold重验证：OOS 300天⊃219天训练窗口 → IC2判定需用681天重新验证
- Score区间过滤：同上
- 失败的研究（score衰减等）不需重新验证——失败结论本身够保守

### 25. IS/OOS效率比作为参数稳健性指标

**定义**：IS/OOS效率比 = IS段平均日PnL / OOS段平均日PnL

**判定标准**：
- 0.8-1.2：参数稳定，可以放心外推
- 1.2-1.5：适度依赖训练窗口，可接受但需监控
- 1.5-2.0：明显依赖，警惕
- 2.0-3.0：严重依赖，警报
- \>3.0：极端警报，参数可能严重过拟合或市场结构质变

**实际案例（2026-04双窗口验证）**：
- IM: IS 219天+10.8pt/天, OOS 681天+7.4pt/天, **比值1.46**（适度）
- IC: IS 219天+9.5pt/天, OOS 681天+2.4pt/天, **比值3.96**（极端警报）

**操作意义**：
- 比值>2的品种，所有"近期发现"都需要带保守标签
- 优先做"两品种都有效"的优化（对IC近期依赖不敏感）
- 每个新研究必须报告IS/OOS效率比作为"可外推性"标签
- IC比值3.96意味着IC的参数可能过拟合到最近219天，或者IC市场结构发生了真实变化——需要更多证据区分

### 26. 信号的物理双重性

**定义**：一个统计上相关性很强的信号（r>0.5），在物理上可能对应多种相反的事件。如果在所有可观测特征上都无法事前区分，信号只能在分布层面利用，无法转化为单点决策规则。

**实际案例：M分衰减信号（2026-04 三轮研究）**

第一轮（总score衰减）：r=-0.47/-0.52 → 失败。应触发/误触发不可分。
第二轮（M子分量衰减）：r=-0.51/-0.57，双窗口完美一致。B.6揭示根因：触发trade里避免STOP_LOSS亏损+6458pt和错过TREND_COMPLETE盈利-6082pt几乎完全对冲。
第三轮（M衰减+V/Q配合区分）：
- "M衰减+V不衰减"组的应出场占比79%
- "M衰减+V严重衰减"组的应出场占比78%
- 差异<1%，V/Q完全无法区分"动量暂停后继续"和"动量真正反转"

**物理本质**：M分衰减="动量消失"，但动量消失有两种相反后果：(a)暂停后趋势继续 (b)趋势真正反转。"暂停后是否继续"取决于更深层的市场结构（资金流、盘口深度、订单流），不是signal_v2当前维度能捕捉的。

**诊断流程**（信号无法转化为规则时按顺序排查）：
1. 阈值选择不对（最浅）
2. 规则形式不对（立即出场 vs 收紧保护）
3. 子分量拆解（聚合信号 vs 单一子分量）
4. 多维度组合（找互补的区分维度）
5. 物理本质不可分（最深，需要全新数据维度）

如果1-4都试过失败，是5的可能性。此时：
- 接受"分布层面真，个体层面不可操作"
- 信号用作风险标签（仓位/止损调整）而非硬触发
- 或引入全新数据维度（盘口、订单流等）——大研究

**跟原则21/22的关系**：
- 原则21：r不够强 → 第一层障碍
- 原则22：r够强但不可分 → 第二层
- 原则26：r够强、部分可分但物理本质双重 → 第三层（最深）

## 研究流程checklist（更新版）

开始新因子/策略研究前过一遍：

**研究设计阶段：**
1. [ ] 初始评估数据量>=500笔样本或1年以上？
2. [ ] 是否保留>=30%数据作为纯观察数据不参与假设形成？
3. [ ] 假设是否有先验经济/微观结构解释？还是纯数据挖掘？
4. [ ] **开始参数搜索前**：是否做了基本面分析，算了理论最大空间？
5. [ ] 参数空间是否有限（<20个组合）？是否设置了"高原vs尖峰"判定？
6. [ ] forward_periods是否覆盖了合理的兑现窗口范围？

**评估阶段：**
7. [ ] 评估指标是否包括IC和分组bps差两个维度？
8. [ ] 初始发现看起来很强时，是否计划在独立数据上做OOS验证？
9. [ ] **看到反直觉结果时**：是否先查了代码和定义bug，再找市场解释？
10. [ ] **评估结果时**：是否多维度综合判断，避免单一硬门槛？

**复杂规则阶段：**
11. [ ] **引入复杂规则前**：先问"更简单的版本行不行"？
12. [ ] **sanity check**：在至少3个代表性日子上手动走一遍？（慢速趋势、快速反转、止损日）
13. [ ] **并行机制**：路径间的优先级和互斥关系是否明确？使用率是否均衡？
14. [ ] 规则中的每个magic number是否都当作参数计入自由度？

**多进程与per-symbol阶段：**
15. [ ] **多进程sweep**：参数是否通过args传给子进程？（不能依赖主进程修改全局变量）
16. [ ] **sweep结果检查**：不同参数值的结果是否真的不同？（完全相同=bug）
17. [ ] **IS/OOS反转检查**：IS最优在OOS上是否仍然最优？（反转=不上线）
18. [ ] **跨品种复制检查**：是否假设了"A品种有效→B品种也有效"？（每个品种独立验证）

**信号→规则转化阶段：**
19. [ ] **转化效率事前估算**：所需正确率 P/(P+L) 是多少？相关性能支撑吗？
20. [ ] **两类拆解**：应该触发 vs 误触发在可观测特征上能否区分？（不可分=赌博）
21. [ ] **负面结果尽职调查**：基线一致性、内部对账、子组拆解、副产品识别
22. [ ] **IS/OOS不重叠**：OOS段是否包含训练数据？IS是否跟训练窗口对齐？

全部肯定回答=流程合格。任何一条缺失=风险点。

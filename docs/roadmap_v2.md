### 1.6 振幅切换动态止损研究（2026-04-08）

**动机**：动态lb用振幅二值切换成功（+20%），止损能否同理？低振幅日用更紧止损快止血，高振幅日用更宽止损给趋势空间。

**IM 2D sweep（lo × hi，振幅阈值1.5%）**：
```
  lo/hi | hi=0.2% | hi=0.3% | hi=0.4% | hi=0.5% | hi=0.6%
  0.20% |  +2831  |  +2979  |  +2836  |  +2894  |  +3242
  0.30% |  +2740  | +2888 ◀ |  +2744  |  +2803  |  +3151
```
- IM最优 lo=0.2% hi=0.6% (+354pt)：分半❌(前半-133)、单日❌(60%) → 不实施
- **IM固定0.3%仍是最优**：hi=0.3%列全面最优，高振幅日放宽反而亏（IM高波动日需要紧止损快止血快重入）
- lo=0.3% hi=0.4% = +2744（-144pt），证实IM不应放宽

**IC 2D sweep**：
```
  最优: lo=0.4% hi=0.5% = +3086pt (Δ+197)
  次优: lo=0.4% hi=0.55% = +3039pt (Δ+151)
```
- 稳健性三检（放宽邻域至80%通过率）：
  - (0.4%, 0.5%)：邻域55%❌、分半✅(+113/+84)、单日39%❌
  - (0.4%, 0.55%)：邻域68%❌、分半✅(+113/+37)、单日52%❌
- **分半均通过**（改善方向正确），但单日集中（20251120单日+78pt占39-52%）
- 逻辑成立：IC低振幅日0.4%比0.5%更优，高振幅日保持0.5%

**品种止损特性对比**：
- IM：高波动日需要**紧**止损（0.3%固定最优，放宽到0.4%反而-144pt）
- IC：高波动日需要**宽**止损（0.5%），低波动日可以收紧到0.4%
- 原因：IM波动率较小，紧止损后能快速重新入场抓新趋势；IC波动率更大，紧止损被正常回撤震出

**结论**：暂不实施。等400+天数据后重新验证IC动态止损（lo=0.4% hi=0.5%），届时高振幅日样本从14天增至30+天，统计更可靠。

### 1.9 Q分重构：历史同时段分位数法（2026-04-09，已实施）

**根因**：旧Q分用rolling 20-bar均值，跨日时被昨天尾盘低量拉低（开盘虚高Q=20），盘中被今天开盘高量拉高（中盘虚低Q=0）。全天43%的bar Q分评估不正确。

**新算法**：
```
hist_vols = 过去20个交易日同一UTC时段的volume列表
percentile = sum(v <= today_vol for v in hist_vols) / len(hist_vols)
Q = 20 if pct > 0.75 else (10 if pct > 0.25 else 0)
```

**新旧Q分布对比**：
| 时段 | 旧Q=20 | 旧Q=0 | 新Q=20 | 新Q=0 |
|------|--------|-------|--------|-------|
| 开盘 | **66%** | 0% | 29% | 30% |
| 盘中 | **0%** | 33% | 28% | 29% |
| 尾盘 | 12% | **0%** | 27% | 29% |

新Q在三个时段分布均匀（~29%/44%/27%），消除了系统性偏差。

**新baseline**：
- IM: +2237pt（旧+2182，+55pt +3%）
- IC: +2428pt（旧+2935，-507pt -17%）
- 合计: +4665pt（旧+5117，-452pt -9%）
- IC下降原因：旧Q在盘中永远给Q=10，新Q有时给Q=0→部分信号低于阈值不触发
- 这是更真实的baseline（消除了跨日偏差的虚假利润）

**实现**：
- `compute_volume_profile(bar_5m_all, before_date, lookback_days=20)` 构建历史profile
- `volume_percentile_q(vol, hist_vols)` 计算分位数Q

## 1.4 Phase 1.5：Slope Reversal + 日内均值回归（2026-04-15 新增）

### 1.5.2 环境识别 + 策略模式切换（下一步优先）

**目标**：构建"波动率区间过滤器"作为动量/均值回归的总开关。

**核心理论支撑**（日内交易适用性评估）：
- **日内季节性**：早盘(09:30-11:00)动量效应强，午后(13:30-14:50)均值回归效应强
  - 完美解释为什么v2动量策略在震荡日亏钱：早盘无趋势追高杀低，午后有回归但v2抓不到
- **日内波动率过滤器**：日线级标准差太迟钝，应改用开盘30min Parkinson/GK波动率判断趋势日vs震荡日
  - 我们的振幅过滤(check_low_amplitude)已是这一思路的简化版
- **均值回归激活时段**：集中在午后，或尾盘VWAP偏离极值点
- **硬止损必须**：日内均值回归绝不允许无限摊平，固定ATR倍数止损

**已有资产**：
- Hurst指数60d：已在daily_model_output中，当前仅显示不参与评分
- 开盘30min振幅：r=0.43最强PnL预测因子，已用于振幅过滤
- 震荡日识别：amp<1.25% + MA穿越≥2 + 偏离<0.8% + 对称>0.25（Phase 1.11）
- ATR短/长比：V分已使用
- VWAP偏离度：已记录在signal_log（r(M)=0.67冗余，但作为均值回归锚点有独立价值）

**设计方案**：
1. **日频regime（Hurst驱动）**：
   - H < 0.45 → 震荡期：激活均值回归模式，抑制动量追单
   - H > 0.55 → 趋势期：保持v2动量模式
   - 0.45-0.55 → 中性期：两者共存
2. **日内regime（开盘30min分类）**：
   - 振幅 < 1.0% + Parkinson vol低 → 震荡日，午后激活均值回归
   - 振幅 > 1.5% → 趋势日，全天v2动量模式
   - 中间区域 → 上午动量，午后允许均值回归
3. **均值回归入场信号（午后时段，震荡regime下）**：
   - VWAP偏离度极值反转：price偏离VWAP > 2σ时反向开仓，回归VWAP平仓
   - Z-Score极值反转：5min Z-Score（60-180min窗口）< -1.8做多，> +1.8做空
   - 布林带2σ外轨触碰后回归中轨
   - **时间止损**：日内半衰期约14min，持仓超30min未回归则强制平仓
   - **硬止损**：固定ATR倍数，绝不摊平
4. **时段分工**：
   - 09:30-11:00：v2动量策略主导（早盘动量效应强）
   - 13:00-14:50：震荡日切换为均值回归（午后回归效应强），趋势日继续v2

**验证计划**：
- 先在Phase 1.11识别的62天IM震荡日上测试午后Z-Score/VWAP均值回归
- 对比：震荡日用均值回归 vs 震荡日用动量（当前-798pt）vs 震荡日空仓
- 手续费和滑点必须纳入（日内均值回归利润空间窄）

    """聚合为5分钟级微观结构指标，与index_min时间对齐"""
    return DataFrame with columns:

**数据覆盖**：IM从2022-07开始，覆盖216天全部回测日。

### 2.2 OFI（订单流不平衡）因子

经典OFI定义（Cont, Kukanov & Stoikov, 2014）：
```python
# 每个tick：
#   bid_price上升 → bid贡献 = +Δbid_volume
#   bid_price下降 → bid贡献 = -bid_volume_prev
#   ask_price下降 → ask贡献 = +Δask_volume
#   ask_price上升 → ask贡献 = -ask_volume_prev
# OFI_5min = Σ(bid贡献 - ask贡献)
```

**接入方式**：
- 先跑因子评估（IC/Daily IC/分组收益/regime分组）
- 和M/V/Q的相关性必须<0.5
- 通过后作为新的O分（0-15分），或替代/增强Q分

### 2.3 盘口不平衡（OBI）因子

```python
OBI = mean over 5min of [(bid1_volume - ask1_volume) / (bid1_volume + ask1_volume)]
```

更简单的即时买卖压力指标。A股期货盘口深度较浅，噪音可能大，需要验证信噪比。

### 2.4 成交强度（Trade Intensity）

```python
trade_intensity = count(ticks in 5min) / rolling_mean(count, 48)  # vs 4小时均值
```

和Q分（成交量）的区别：同样1万手成交，100笔大单 vs 10000笔小单含义不同。

### 2.5 大单占比

```python
# 先统计P95阈值：SELECT PERCENTILE(volume_delta, 0.95) FROM tick WHERE ...
large_trade_pct = sum(vol WHERE single_trade_vol > P95) / total_vol
```

### Phase 2 预期与风险

- 乐观：接入后+5-10%增量
- 风险：tick噪音大，OFI近似误差可能导致因子无效
- **关键里程碑**：2.1 Pipeline完成后，2.2-2.5可并行评估

## 十一、工程方法论

### 新因子接入检查清单

1. **独立性**：vs M/V/Q Pearson相关性 < 0.5
2. **预测力**：按因子值分4组，各组WR和avg PnL有显著差异
3. **单日验证**：用已知受影响日期确认因子值变化符合预期
4. **Patch正确性**：确认修改的变量是score_all实际调用的
5. **216天回测**：加入后PnL > baseline
6. **Sensitivity**：参数±20%结果稳健

### 关键教训

1. monkey-patch了错误变量（`_TIME_WEIGHTS` vs `session_multiplier`）→ 假阳性
2. 事后模拟（乘在最终score上）≠ 真实集成（加进raw_total）→ 结论方向有偏差
3. "有益偏差"不应修复（跨日GAP、forming bar Q=0）→ 修复后反而亏钱
4. 现货/期货价格混用是致命bug → 持仓管理必须统一价格源
5. 时间标签对齐影响全部时间相关逻辑 → 永远用`label='left', closed='left'`
6. SYMBOL_PROFILES参数只在代码真正读取时才生效 → 修复前确认读取路径
7. 稳健性三检（时间分段+参数邻域+单日贡献）三项全过才实施
8. 日频期权指标偏相关<0.15 → 不等于5分钟级别也无效，但要有预期管理

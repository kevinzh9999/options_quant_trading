# 项目计划：日线波段策略

> 目标：在日内策略的"盲区"（低振幅单边趋势期）赚取多日持仓利润。与日内策略互补。
>
> 创建：2026-04-04

---

## 核心理念

**日内策略亏钱的日子 = 波段策略赚钱的日子。**

215天数据验证：
- 日内策略在两段单边趋势（6/23~8/25 +23%, 12/17~1/12 +14.7%）只捕获了7%的涨幅
- 理想多日持仓可捕获：+1,400pt + +1,068pt = +2,468pt
- 这两段如果能捕获50%，就是 +1,234pt 增量

| 市场状态 | 日内策略 | 波段策略 | 识别方法 |
|----------|---------|---------|---------|
| 高振幅震荡 | +49/天 | 不参与 | 日内振幅>1.5% |
| 低振幅趋势 | -27/天 | 赚钱 | 日内振幅<1.0% + 方向明确 |
| 中等振幅 | +2/天 | 按信号 | 1.0-1.5% |

---

## 需要谨慎的实操问题

### 1. 隔夜gap风险（Phase 1必须量化）
"理想+2,468pt"没有扣除隔夜gap损失。A股期货有涨跌停板但gap可以很大。
- Phase 1必须统计：波段持仓期间每天的隔夜gap（开盘价-昨收盘价）
- 如果gap损失吃掉大部分趋势利润，策略不可行

### 2. 日内策略暂停的机会成本（Phase 1必须量化）
波段持仓期间如果暂停日内策略 → 波段亏钱的日子日内也被暂停 → 双重损失。
- Phase 1必须统计：日内策略在"趋势期"的PnL是多少
- 这个数字是波段策略必须覆盖的底线

### 3. 锁仓成本被低估
锁仓占用双倍保证金（多空各一手）。以IM为例：
- 锁仓2手保证金 ≈ 60万
- 每次roll手续费 ≈ 300元 + 滑点
- 5天持仓roll 4次 ≈ 1,200元 + 滑点
- 对比预期收益需评估是否可接受

### 4. 入场信号可能太滞后
"连续3天同方向 + ADX>25"——确认趋势时可能已走了30-50%。
- 1月那波7300→8400：第3天确认时可能已到7600（+300pt），只能吃后半段800pt
- 后半段恰好是反转风险最高的时候
- Phase 1需统计：趋势确认时已走了多少比例

## Phase 1：趋势识别研究 + 可行性验证（1-2天）⬅ 优先执行

**目标**：回答一个核心问题——在1月那种单边涨的月份，如果第3天开盘买入持有到ADX回落，扣除隔夜gap后能赚多少？如果答案是"比日内策略在那个月的-145pt好不了多少"，项目优先级降低。

### 1.1 日线技术指标回算

用 `index_daily`（2015~2026）计算：
- 20日均线（MA20）
- ADX(14)
- 日线振幅
- 隔夜gap = open[t] - close[t-1]

### 1.2 定义"趋势期"

候选定义（需回测验证哪个最优）：

```python
# 方案A：MA + 方向一致性
trend_up = close > MA20 连续 N 天 AND close[today] > close[today-N]

# 方案B：ADX + 方向
trend = ADX > 25 AND (DI+ > DI- 连续 N 天)

# 方案C：日线振幅收窄 + 方向
trend = 5日平均日线振幅 < 1.2% AND |5日累计涨跌| > 3%

# 方案D：组合
trend = (close > MA20 连续 3天) AND (ADX > 20) AND (5日振幅 < 1.5%)
```

### 1.3 历史趋势期统计

对2015~2026的趋势期统计：
- 持续天数分布
- 总涨跌幅分布
- 最大回撤
- 趋势结束后的走势（反转概率）

### 1.4 关键可行性验证（Phase 1的核心）

#### a) 隔夜gap量化
```python
# 对每个趋势期，统计每天的隔夜gap
# gap_pnl = sum(open[t] - close[t-1]) for all days in trend
# 趋势利润 = close[-1] - entry_price
# 净利润 = 趋势利润 - |不利gap累计|
```

#### b) 日内策略机会成本
```python
# 对215天中被识别为"趋势期"的日子
# 统计这些日子的日内策略PnL（从已有回测数据）
# 这是波段策略必须超过的底线
intraday_pnl_during_trend = sum(daily_pnl[d] for d in trend_days)
# 波段净收益 必须 > intraday_pnl_during_trend
```

#### c) 入场滞后量化
```python
# 对每个趋势期：
# 趋势起点 = 第1天close
# 确认入场 = 第N天open（N=3或ADX>25的第一天）
# 已错过 = (entry - trend_start) / (trend_end - trend_start)
# 如果平均已错过 > 50%，入场太晚
```

#### d) 1月案例具体验算
```python
# 12/17~1/12 单边上行 +14.7%
# 假设第3天(12/19)开盘买入：entry ≈ 7330
# 假设ADX<20退出(约1/14)：exit ≈ 8240
# 理论利润 = 8240 - 7330 = 910pt
# 扣除隔夜gap损失
# 扣除锁仓roll成本（约4次×300=1200元≈6pt）
# 净利润 vs 日内策略同期-145pt
```

### 1.5 决策点

Phase 1结束后做Go/No-Go决策：
- **Go**：净利润（扣gap+成本）显著优于日内策略机会成本 → 继续Phase 2
- **No-Go**：净利润接近或低于日内策略机会成本 → 停止，专注优化日内策略
- **Partial Go**：只在极端单边月份（如连涨>5天）手动持仓，不做系统化策略

## Phase 2：入场/出场规则设计（1-2天）— 仅Phase 1通过后执行

### 2.1 入场信号

```python
entry_long = (
    close > MA20                      # 价格在均线上方
    AND close > close[-3]             # 3天累计上涨
    AND ADX > 20 AND ADX_rising       # 趋势正在加强
    AND avg_daily_range(5) < 1.5%     # 日内振幅不大（排除震荡日）
)
```

### 2.2 仓位管理

- 1手IM或IC（验证期间）
- 锁仓持仓（避免平今手续费万分之2.3）
- 次日开盘平昨仓+开新仓（roll），手续费万分之0.23
- 注意：锁仓2手保证金≈60万，需确认账户余量

### 2.3 出场规则

| 优先级 | 出场条件 | 说明 |
|--------|---------|------|
| 1 | 日线收盘跌破MA20 | 趋势结束 |
| 2 | ADX从>25回落至<18 | 趋势减弱 |
| 3 | 单日亏损>1.5% | 固定止损 |
| 4 | 持仓满10天 | 时间止损（防死扛） |
| 5 | 日内振幅突然>2.5% | 市场转为震荡，切回日内策略 |

### 2.4 与日内策略协调

**关键原则：波段持仓期间日内策略不暂停。**

两个策略独立运行，通过仓位限制协调：
- 波段持仓1手 + 日内最多1手 = 总仓位≤2手
- 日内策略的锁仓和波段策略的锁仓互不干扰
- 如果波段方向和日内信号一致 → 加仓效果
- 如果反向 → 天然对冲

这样避免了"暂停日内策略"带来的机会成本问题。

## 风险

1. **隔夜gap可能吃掉大部分利润**（Phase 1验证）
2. **入场信号滞后**：趋势确认时可能已走了30-50%，剩余空间有限
3. **A股期货趋势持续性**：可能不如海外CTA标的，频繁假突破
4. **锁仓占保证金**：波段+日内同时需要更多保证金（≈120万/IM双手）
5. **回测偏差**：日线回测无法反映盘中的止损精度

## 成功标准

- Phase 1：1月案例净利润（扣gap）显著优于日内策略同期的-145pt
- Phase 1：趋势确认时平均已错过比例 < 40%
- Phase 3：波段策略在11年中至少8年盈利
- Phase 3：联合回测（日内+波段）比单独日内在215天上多 > 500pt

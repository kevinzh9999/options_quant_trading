## 待研究方向

### 优先级1：打开已有但被归一化的信号（代码已有，只需开关+验证）

**1.1 daily_mult (dm)**
- 当前：固定1.0
- 原设计：顺势dm=1.1，逆势dm=0.9（IM/IC），IF保持1.0/1.0
- 数据源：index_daily前一日close vs 前5日trend
- 测试方法：run_day()中打开dm计算（不归一化daily_bar），IS/OOS验证
- 历史教训：之前d_override(briefing)导致-19.2%劣化才被禁用，但算法dm跟briefing dm不同
- 预期：dm本身只有±10%的乘数效应，改善可能有限

**1.2 intraday_filter (f)**
- 当前：固定1.0（因daily_bar=None）
- 原设计：日内涨跌幅>2%逆势→打折0.3-0.5，顺势不打折
- 数据源：当前price vs 昨收close
- 测试方法：传入real daily_bar到run_day()
- 注意：intraday_filter跟dm耦合（都需要daily_bar），可能需要一起测

**1.3 zscore过滤**
- 当前：固定None
- 原设计：|z|>2硬过滤（极端偏离时不开仓），|z|>1.5软打折
- 数据源：index_daily近30日close → EMA20/STD20
- 测试方法：传入real zscore到run_day()
- 预期：z-score过滤是保守策略（减少极端日交易），可能减少笔数但提高胜率

**1.4 is_high_vol regime切换**
- 当前：固定True
- 原设计：GARCH forecast vol / 长期均值 > 1.2 → high vol模式
- 影响：high vol用不同的intraday_filter阈值
- 测试：传入实际GARCH regime

### 优先级2：重设计触发率极低的因子

**2.1 B因子（布林带突破）放宽条件**
- 当前触发率：0%
- 瓶颈："前5根全在中轨同侧"太严格
- 候选改进：
  - 放宽到"前3根同侧"
  - 只要求"前5根中4根在同侧"
  - 去掉放量/窄带附加条件，只保留突破判断
- 风险：放宽太多可能引入噪音

**2.2 S因子（趋势启动）放宽条件**
- 当前触发率：3%
- 瓶颈："振幅>均值×1.5"要求太高
- 候选改进：
  - 降到1.2x或1.3x
  - 放量阈值从60%降到50%
  - 改为"三条件满足两条即可"
- 风险：同上

**2.3 RSI bonus激活**
- 当前：需zscore≠None才生效，因zscore=None永远不触发
- 修复：解耦RSI和zscore的依赖关系
- 独立RSI过滤：RSI<25做多加分，RSI>75做空加分

### 优先级3：引入新数据维度

**3.1 Hurst指数作为regime过滤**
- 数据：daily_model_output hurst_60d
- 用法：H>0.55趋势期→保持当前策略，H<0.45震荡期→提高threshold或不交易
- 测试方法：按Hurst分桶看PnL差异

**3.2 季月贴水变化率**
- 数据：daily_model_output discount_rate_iml2/iml3
- 用法：贴水扩张（恶化）→偏空信号加分，贴水收缩→偏多信号加分
- 需要先验证贴水变化率与日内PnL的相关性

**3.3 VRP方向指导**
- 数据：daily_model_output vrp
- 用法：正VRP（IV>RV）→ 卖波动率环境，可能波动率会降→影响方向判断
- 需要验证VRP与日内方向PnL的关系

**3.4 ADX趋势强度过滤**
- 数据：bar_5m实时计算ADX(14)
- 用法：ADX<20时信号减分（无趋势），ADX>35时加分（强趋势）
- CLAUDE.md记录：待100+笔ADX>35样本后验证硬过滤

### 优先级4：Tick微观结构（工作量大）

**4.1 买卖盘不平衡**
- 数据：futures_tick表
- 指标：近N秒的买盘成交量/卖盘成交量比值
- 用法：>1.3看多偏向，<0.7看空偏向，与M方向确认

**4.2 大单频率**
- 数据：futures_tick表
- 指标：近N分钟内大单（>均单×3）的频率和方向
- 用法：大单方向与M方向一致→加分

**4.3 成交速度**
- 数据：futures_tick表
- 指标：单位时间成交笔数的变化率
- 用法：成交加速→趋势延续信号

### 优先级5：期权隐含信息（需额外计算）

**5.1 日内IV变化**
- 数据：options_min表
- 指标：ATM IV在日内的变化方向和速度
- 用法：IV急升→恐慌→做空可能更好

**5.2 日内PCR变化**
- 数据：options_min表
- 指标：Put/Call成交量比值的日内变化
- 用法：PCR上升→看跌情绪增加

### 优先级0（最高）：M分拐点检测 + entry/exit统一

**核心问题**：M用lb根bar的net change判断方向，在趋势拐点滞后。
统计上M大小与PnL相关性≈0，实盘上M方向跟不上反转。

**关键洞察（2026-04-14复盘）**：M拐点检测不只是entry问题，也是exit问题。
如果M能判断拐点：
- exit有了依据（当前方向的趋势结束 → 平仓）
- entry有了依据（新方向确认 → 反手开仓）
- 这是吃波动的一体两面，entry和exit应该用同一套拐点检测

**候选拐点检测指标**：

1. **RSI极值**（最简单）
   - RSI>75 + M=LONG → 超买，不追多/考虑平多
   - RSI<25 + M=SHORT → 超卖，不追空/考虑平空
   - 优点：已在计算，无新参数
   - 风险：强趋势日RSI可持续极值不反转

2. **布林带位置**（与exit对称）
   - 价格在UP+区域 → 趋势可能到头
   - 价格从上轨回落穿越中轨 → 反转确认
   - exit已用布林带（TREND_COMPLETE），entry也用则逻辑统一
   - 风险：窄带震荡时来回穿越

3. **M减速度**（不需新指标）
   - M连续2-3根bar下降（50→40→30）→ 趋势衰弱
   - 用法：M减速时降低当前方向的置信度
   - 优点：只跟踪M自身变化率
   - 风险：正常波动也有M波动

4. **成交量极端**
   - 量能枯竭（Q从高→低）+ M保持同向 = 趋势乏力
   - 放量反转（Q突然升高但方向变了）= 拐点确认

5. **TREND_COMPLETE反向信号**
   - TREND_COMPLETE判定"旧趋势完成" → 反方向entry的前置条件
   - 不能简单反手（可能继续原方向），但可以作为"准备反向"的触发器

**推荐组合**：RSI极值 + 布林带位置
- 两个条件同时满足时 = 高置信度拐点信号
- entry不追极端，exit在到达极端时平仓 → 同一套"极端检测"
- 物理直觉：价格在布林带极端 + RSI超买超卖 = 不适合追趋势

**entry/exit统一框架设想**：
```
拐点检测器（RSI + 布林带 + M减速）
  → entry侧：不追极端方向，等反转确认后入场
  → exit侧：当前持仓方向的拐点 = 平仓信号
  → 反手：平仓信号 + 反向M确认 = 反手入场
  
例：今天13:35 IM
  持仓SHORT，价格8060触及布林下轨，RSI=24
  → 拐点检测器发出"空头趋势可能结束"
  → exit：平空（而不是等TIME_STOP 65min后才平）
  → entry：等M切换到LONG → 反手开多
  → 捕捉8060→8140的80pt上涨
```

## 方法论约束

1. **IS/OOS验证**：IS=20250101~20260413, OOS=之前
2. **用run_day()做backtest**：不自建loop，用score_transform回调
3. **post-hoc标注 vs 重跑**：
   - Entry评分变体 → score_transform回调（改threshold判断）
   - Exit规则变体 → 必须monkey-patch或参数化重跑
   - 新过滤条件（如Hurst regime）→ 可以post-hoc分析先看潜力，再重跑验证
4. **baseline一致性**：所有对比必须用同一个run_day()跑出的baseline
5. **不过拟合**：每个改进最多2个参数，IS改善+OOS改善都要求为正

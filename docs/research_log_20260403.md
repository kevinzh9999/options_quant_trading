# 研究日志 2026-04-03：ME退出与score确认机制研究

## 背景

4/2复盘发现：ME在score仍高时退出，冷却期后追入趋势末端亏损。例如IC 13:50 ME退出（score=87），15分钟后14:05追入，-29pt止损。问题不在"追不追"，而在"该不该退"。

---

## 一、问题分析

### 核心矛盾

Entry系统（score_all）和Exit系统（check_exit）看不同时间尺度：
- **ME**：3根bar窄幅（15分钟微观）→ "动量耗尽"
- **Score**：12根bar动量+ATR+成交量（60分钟宏观）→ "趋势还在"

当两者冲突时（ME说退、score说留），当前系统无条件听exit。

### 4/2具体案例

**IC #4→#5**：
- 13:50 ME退出 @7586（score=87，远超阈值65）→ +36pt
- 15min冷却
- 14:05 追入 @7580（score=78）→ 14:20 TRAILING_STOP -29pt
- 净效果：如果不退出，13:05→14:20持有到反转，+14pt（好于+36pt-29pt=+7pt）

**IM #4→#5**：
- 13:50 ME退出 @7605（score=72，超阈值60）→ +35pt
- 14:05 追入 @7596 → 14:20 STOP_LOSS -38pt
- 净效果：-3pt（远差于如果不退的+22pt）

### 退出类型分类

| Exit类型 | 检测内容 | 时间尺度 | 适合score确认？ |
|---|---|---|---|
| STOP_LOSS | 亏损超限 | 价格事实 | **绝不**（安全阀） |
| EOD_CLOSE | 收盘 | 规则性 | **绝不** |
| LUNCH_CLOSE | 午休亏损 | 规则性 | **绝不** |
| TRAILING_STOP | 利润回撤 | 价格事实 | **不应该** |
| TIME_STOP | 60min无盈利 | 时间 | **不应该** |
| **ME** | 3根窄幅+15m极端 | 15分钟微观 | **最适合** |
| **TC** | 5m+15m双极端 | 双周期 | **可能适合** |
| MID_BREAK | 破中轨+15m确认 | 结构性转折 | **不太适合** |

## 二、Score确认方案

### 逻辑

ME/TC触发时，检查prev_score（上一根bar的评分，无lookahead）。如果score仍高于阈值，说明趋势宏观结构未变，不退出。

```python
if reason in ("MOMENTUM_EXHAUSTED", "TREND_COMPLETE"):
    if prev_score >= threshold and prev_direction == position_direction:
        continue_holding  # 趋势未完，暂不退出
    else:
        do_exit  # score也确认弱了，退出
```

### 34天回测结果

| 方案 | IM+IC PnL | vs baseline | 笔数 | 均PnL/笔 |
|---|---|---|---|---|
| Baseline | +1382 | — | 301 | 4.6pt |
| Score确认(ME+TC+MID) | +1512 | **+130 (+9.4%)** | 175 | 8.6pt |

### ME被阻止的统计

| 品种 | 阻止次数 | 改善率 | 平均额外PnL |
|---|---|---|---|
| IM | 122次 | 61% | +4.8pt/次 |
| IC | 101次 | 72% | +13.4pt/次 |

### 分解：哪种exit适合score确认

| 方案 | IM+IC PnL | vs baseline |
|---|---|---|
| 只ME | +1467 | +85 |
| ME+TC | +1539 | **+156** ← 最好 |
| ME+TC+MID | +1512 | +130（MID_BREAK不该阻止，-26pt） |

## 三、稳健性验证（不通过）

| 测试 | 结果 | 判定 |
|---|---|---|
| 时间分段 | 前半段 -58pt，后半段 +187pt | ❌ 不一致 |
| 每周 | 4/5周改善，但W5贡献134% | ⚠ 勉强 |
| 阈值敏感 | 系数0.8已变负(-50pt)，1.2暴跌(-509pt) | ❌ 依赖精确阈值 |
| 单日集中 | 最大单日53%，仅14/34天改善 | ❌ 少数天驱动 |

### 结论：不实施

+130pt的改善来自3月26日一天的+68pt（占53%）。阈值从1.0偏移到0.8就变亏钱。不够稳健。

## 四、深层问题（4/3分析）

### 4/2详细回放揭示的问题

**IC score确认版 #2**：10:10 S@7675 → 13:00 @7621 +54pt（170min）

上午表现极好——6次ME/TC被阻止，持仓穿越午休。但13:00退出的原因是：
- 午休后第一根bar score从67降到<65（刚好穿过阈值）
- ME再次触发，这次不阻断 → 退出

**问题**：
1. **午休造成score技术性重置**——不是趋势结束，是指标跨午休断档需要"热身"
2. **二值阈值太脆弱**——score=64退、score=66不退，一分之差决定放弃+54pt盈利仓位
3. 退出后13:15重新开仓@7606（比13:00的7621差了15pt入场价）

### 可能的改进方向（待数据积累后研究）

1. **Score变化趋势替代二值阈值**：要求连续N根bar score都在下降才确认退出，而非单bar<阈值
2. **更大buffer**：ME阻断后要求score降到阈值×0.7才放行（64 vs 45.5的差距大得多）
3. **午休热身期**：午后前2根bar不做ME/TC判断（让指标重新稳定）
4. **时间衰减**：持仓时间越长，阻断条件越严格（长持仓=趋势已充分发展，退出概率应提高）
5. **只对ME做score确认**（不含TC/MID_BREAK）：ME单独贡献+85pt，且ME的微观判断最容易和宏观趋势冲突

### 数据需求

当前34天数据不足以区分"稳健改善"和"过拟合"。需要：
- 至少100天回测数据（当前34天）
- 包含不同市场环境（趋势日+震荡日+反转日）
- 特别需要"ME正确退出"的反例（趋势真的在ME时结束，score滞后给出高分）

## 五、Position Manager与Shadow机制确认

### 问题：executor拒绝信号后position_mgr占位

Monitor发信号 → position_mgr占位 + shadow注册 → executor拒绝 → position_mgr仍占位

**结论：这是正确的设计。**
- Shadow是信号系统的benchmark（全执行假设），不应因executor拒绝而改变
- position_mgr跟shadow同步，确保信号频率与回测一致
- 实盘PnL vs shadow PnL的差距 = 操作者决策的净贡献（可正可负）
- Shadow PnL不是"上限"——操作者正确拒绝亏损信号时，实盘可以优于shadow
